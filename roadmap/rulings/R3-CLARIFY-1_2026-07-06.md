## Background — what prompted this ruling

In the 2026-07-06 parallel-audit dispatch, the shared doctrine brief handed to
Lens A / Lens B auditors defined the R3 forbidden-token scan as a raw substring
regex: `/ai|agent|claude|opus|sonnet|gpt|copilot|co-authored-by|generated-by|assistant|computer/i`

Applied literally, this regex matches inside ordinary English words:
- `ai` inside `fail`, `chai`, `aix`, `again`, `against`, `contain`, `maintain`, ...
- `agent` inside `management`, `arrangement`, ...
- `ai` inside `package-lock.json` dependency-funding URL fragments.

The strict reading made the C1 PR (#1 in `tgp-importer-extension`) fail R3 on
`package-lock.json` — a spurious P0 caused by the auditor's regex being wider
than the underlying rule. R3 the rule bans **AI-attribution tokens**; R3 the
regex was catching every English word containing "ai."

## The ruling

R3 forbidden-token scans use **word-boundary-anchored and attribution-shaped**
matches, not raw substrings.

### Canonical scan patterns (case-insensitive except where noted)

Any commit metadata (author, committer, subject, body), any PR title/body,
and any file body committed under this doctrine is a violation if it matches
**any** of the following:

1. `\bAI\b`  — the capitalized acronym `AI` as a standalone token (case-sensitive on this one — matches `AI`, `AI-assisted`, `AI/agent`, `AI's`, etc.; does **not** match `ai` inside `fail`, `chai`, `aix`).
2. `\b(agent|assistant|computer)\b` — only as standalone attribution/authorship terms.
3. `\b(claude|opus|sonnet|haiku|gpt|copilot|gemini|llm|chatgpt|anthropic)\b`.
4. `co-authored-by:` — any casing.
5. `generated-by:` — any casing.
6. `\b(authored|assisted|written|generated)-by-ai\b` — any casing.
7. `\bai-(generated|assisted|written|authored)\b` — any casing.

## Auditor implementation notes

Auditors performing R3 scans should use case-sensitive matching for `\bAI\b`
(so the acronym is caught but lowercase substrings are not) and case-
insensitive for every other pattern. Example shell one-liner for auditor
pre-flight (illustrative, not normative):

```bash
grep -nE '\bAI\b' -- <files>
grep -niE '\b(agent|assistant|computer|claude|opus|sonnet|haiku|gpt|copilot|gemini|llm|chatgpt|anthropic)\b' -- <files>
grep -niE 'co-authored-by:|generated-by:|\bai-(generated|assisted|written|authored)\b|\b(authored|assisted|written|generated)-by-ai\b' -- <files>
```
