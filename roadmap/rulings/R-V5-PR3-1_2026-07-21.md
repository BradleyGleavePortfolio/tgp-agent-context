## Background

The V5 build brief locks the stack `PR-1 → PR-2a ∥ PR-2b → PR-3 → PR-4` and describes PR-3 as a "cross-repo … deterministic end-to-end proof" that is "BUILD (PR-3, tests only)" with "its own R14 dual-lens cycle + git-native `R3_MERGE_RUNBOOK.md` landing." The brief does **not** name (a) the owning repository, (b) the branch/PR name, (c) the exact permitted file paths, or (d) the test-harness location. Per [[R-RULE-AUTHORITY-1_2026-07-20]] an agent must **not invent** that missing specification and must resolve genuine gaps by ruling rather than inference. This ruling supplies the smallest canonical resolution.

Two candidate homes were rejected before landing on the backend:

- **Context-owned orchestration test — REJECTED.** The context repo (`tgp-agent-context`) has **no test runner** and is the docs/state + governance lane (R4). Making it execute a cross-service e2e would require standing up a **new CI workflow** and/or **vendoring** backend-reconstruct and extension-replay product logic into context tests — both forbidden (new workflow; duplication/drift; repo-boundary violation).
- **Multi-repo coordinated PRs — REJECTED.** Driving all three repos in one coordinated landing would require **new cross-repo CI orchestration** and, to exercise the render leg, a **mobile change** — but mobile is a NO-OP for V5 (brief §2.5/§8) unless PR-3 first exposes a contract gap. This is neither the smallest nor rule-compliant.

The backend already owns the reconstruct golden-fixture harness (D1 golden fixture landed in PR #509; the reconstruct path landed in IMPORTER-H #512 / IMPORTER-I #513; brief §8 REUSE lists "golden-fixture harnesses"). The extension replay→emit leg is **already proven and landed** (`test/conformance-alpha.spec.js` at extension `main 95be022`, PR #8 / PR-2b), and the mobile render leg is **already proven and landed** (PR-M4 at mobile `main a5933fd`, a source-neutral contract-1.4.0 consumer). PR-3 therefore reduces to a single backend tests-only PR that certifies both adapters through the unchanged reconstruct core and cites the already-landed extension and mobile legs as evidence.

## The ruling

1. **Owner + kind.** PR-3 is a **tests-only PR in `growth-project-backend`**. The brief's "(cross-repo)" names the *proof span*, not a new repository; the certification is owned by the repo that holds the reconstruct golden. **Context-owned orchestration is expressly rejected** because the context repo has no test runner and PR-3 is defined as an *actual tests-only PR* (a committed artefact that adds the missing byte-identity proof), not an evidence-only run. **Core-diff-zero remains an audit-evidence gate** (brief §4) — it is *proven by* the PR, not *the whole of* the PR: the committed test is what adds the two-adapter, cross-repo byte-identity assertion that no prior PR carries.

2. **Branch name.** `feat/v5-pr3-two-adapter-e2e`.

3. **Permitted paths (backend, test-only — 0 production LOC).** Only `test/**` and `test/fixtures/**` may be touched. Concretely:
   - one replay/reconstruct e2e spec (e.g. `test/scout/v5-pr3-two-adapter-e2e.spec.ts`) that drives BOTH adapters (TrueCoach + `conformance_alpha`) through the **unchanged** reconstruct core and asserts per-family `expect_records`;
   - the backend golden `expect_records` fixtures for both adapters under `test/fixtures/**`;
   - **exactly one SHA256-pinned byte-copy** of the extension emit-side fixture (`test/fixtures/conformance/conformance-alpha.extension.json`), carrying a provenance header pinning extension `main 95be0222df3d47d787566743c8781005d8fbec69` and the sha256 the spec asserts on load.
   - **Forbidden in this PR:** any `src/**`, any DTO, `importer-openapi*` / contract, any migration, any RLS/storage change, any mobile or extension file, any `.github/**` / CI workflow, any `package.json` / lockfile change. Copying **fixture data** (test data) is permitted; importing another repo's **product code** is vendoring and is not.

4. **Cited (not re-run) legs.** The extension replay→emit leg is cited at extension `main 95be022` (PR #8 `test/conformance-alpha.spec.js`); the mobile render leg is cited at mobile `main a5933fd` (PR-M4 #288). PR-3 adds **no** extension or mobile files and re-runs neither.

5. **Required BUILD MATRIX (R124, cross-repo pin).** The PR-3 audit brief and report MUST carry, verbatim:
   ```
   ## BUILD MATRIX
   - backend HEAD: <sha>                # PR-3 head, backend
   - extension HEAD: 95be0222df3d47d787566743c8781005d8fbec69   # PR-2b, landed
   - mobile HEAD: a5933fd6de5616493de75f0db907098b149b955c      # PR-M4, landed
   - ctxrepo HEAD: <sha>
   - PR #<n> head: <sha>
   - PR #<n> base (origin/main): <backend PR-2a tip sha>
   - fixture provenance: extension conformance-alpha.json @ 95be022 sha256=<...>
   - contract: importer-openapi 1.4.0 (byte-identical)
   - timestamp (ISO 8601 UTC): <ts>
   ```
   Any SHA change mid-audit ⇒ `VERDICT: INFRA_DEATH` per R124.

6. **Acceptance commands / evidence.**
   - Backend suite green, including the new two-adapter e2e spec + existing conformance/golden.
   - **core-diff-zero (§4):** `git diff` over the core path-set (engine, DTOs, OpenAPI, migrations, RLS, mobile UI) is **empty**; `grep` of the core (excluding `test/**`) for `conformance_alpha` returns **0**; both adapters' conformance suites green on recorded fixtures.
   - Per-family `staged = reconstructed + skipped + failed` with reasons (billing accounted `excluded`); poison rows honestly `failed`; RLS coach-A cannot read coach-B; contract 1.4.0 byte-identical.
   - The fixture sha256 in the spec equals the pinned extension value.
   - Extension and mobile legs evidenced by citing their landed SHAs (§4 above); not re-executed.

7. **STOP conditions.**
   - Any assertion needs a backend `src/**` change ⇒ **STOP** (that is a core change; adapter mapping belongs to PR-2a, not PR-3).
   - Any contract deviation ≠ 1.4.0 ⇒ **STOP**.
   - Any mobile or extension file change, or any new CI workflow / `.github` / `package.json` / lockfile edit ⇒ **STOP** (out of the PR-3 tests-only scope per [[V5_MULTI_ADAPTER_BUILD_BRIEF]] §6; infra/automation belongs to its own separate lane, outside R14 audit scope for this PR).
   - Fixture sha256 ≠ the pinned extension `95be022` value ⇒ **STOP** (re-pin only via a fresh dated ruling).
   - Backend PR-2a (`conformance_alpha` mapper + golden + conformance test) not GitHub-verified as landed ⇒ **STOP** (PR-3 precondition unmet; verify with `gh pr view --json headRefOid` per [[R-RULE-AUTHORITY-1_2026-07-20]] + R124 before dispatch).

- **Is this the real goal, or a proxy?** Real goal — certify that the reconstruct core is genuinely site-agnostic across two structurally independent adapters end-to-end; PR-3 is the certification, not a proxy metric.
