## Findings

### P2-1 — Read routes not explicitly throttled despite signed-URL signing-cost amplification
- **Where:** `src/community/voice/community-voice.controller.ts` — `list` (`GET .../voice-notes`) and `getOne` (`GET voice-notes/:id`).
- **Detail:** `list` mints a signed download URL **per returned row** (`noteView` → `createSignedDownload`, via `Promise.all`), up to the max page size of 50. Under only the global default authed bucket (~300 req/min/user), a single user can drive up to ~15,000 signing operations/min. The checklist explicitly flags read-side signing-cost amplification as a P2 risk.
- **Mitigating context (why P2, not higher):** reads ARE bounded by the global `UserThrottlerGuard`; the v3-2 classroom sibling controller follows the identical pattern (no explicit `@Throttle` on its GET reads), so this is a consistent pre-existing convention, not a regression introduced here. Signing is to Supabase (not a metered per-call external cost in the audited config).
- **Recommendation:** add an explicit `@Throttle` read bucket (e.g. a `COMMUNITY_READS_PER_MIN` tighter than the global default) on `list`/`getOne`, or memoize/batch signed-download minting per request. Non-blocking.

### P3-1 — `community-voice.e2e.spec.ts` is reflection-only, not a real HTTP integration test
- **Where:** `test/community/voice/community-voice.e2e.spec.ts`.
- **Detail:** Despite the `e2e` name, the suite asserts controller guard/route metadata via reflection and drives the flag guard directly; it does not boot a Nest app or issue supertest HTTP requests. The actual failure-path behaviors (cross-tenant 404, oversized rejection, sub-coach/coach-allowed, flag-off) ARE covered by the **service unit spec** and the **live RLS spec**, so behavior is tested — but no full request→controller→service→DB path is exercised end-to-end.
- **Mitigating context:** mirrors the v3-2 classroom reflection suite convention; coverage of the underlying logic is present elsewhere.
- **Recommendation:** add a thin supertest pass (or rename the file to reflect it is a metadata guardrail). Style/test-depth only.
