### Finding 2 — P2: query-string in metric label + log
```ts
// src/filters/throttler-exception.filter.ts
const method = request.method ?? 'UNKNOWN';
const path = request.url ?? 'unknown';          // <-- request.url INCLUDES "?query=..."
...
this.logger.warn({ message: 'throttler.rejected', userId, ip, path, method, ... });
...
// "Labels are intentionally low-cardinality (method + path); user/ip are in the log."
if (this.metrics) {
  this.metrics.increment('throttler_rejected_total', { method, path });   // unbounded label
}
```
In Express, `req.url` retains the query string; `req.path` does not. A throttled endpoint hit with varying query strings (pagination, tokens, cache-busters) produces a new Prometheus time-series per distinct value — the classic cardinality bomb the very same comment claims to avoid. Query strings can also carry reset/verify tokens, so logging the raw URL risks secret-in-log.
