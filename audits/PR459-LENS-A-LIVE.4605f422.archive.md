### R72-r2 — redactLiterals dollar-quote masking (src/observability/db-stats.service.ts) — PASS
Verified the regex `\$([A-Za-z0-9_]*)\$[\s\S]*?\$\1\$` behavior empirically:
- `$$secret@x.com$$` → `$?$` (anonymous dollar-quote masked). PASS.
- `$tag$user@x.com$tag$` → `$?$` (tagged dollar-quote masked, tag/body do not leak). PASS.
- Prepared-statement placeholders `$1`, `$2`, `$3` are NOT masked — the dollar-quote regex requires a matching closing `$\1$` which a bare `$1` lacks, and the `\d{2,}` pass does not fire on a single digit. Correctness invariant HOLDS for single-digit placeholders. PASS.
- Non-greedy body (`*?`) prevents merging adjacent literals: `$$a$$, $$b$$` → `$?$, $?$`. PASS.
- A fake `$1` inside a dollar-quoted body is correctly swallowed by the outer literal. PASS.
- Masking order (dollar-quote → single-quote → double-quote → digits) ensures the full delimited literal is removed before inner content could leak. PASS.

Observation (informational, NOT a finding): multi-digit placeholders `$10`, `$11`, `$12` have their index digits collapsed by the `\d{2,}` pass (`$10` → `$?`). This is a cosmetic loss of preview fidelity for placeholder indices ≥ 10; it does NOT leak PII and does NOT violate the security invariant. The task's stated invariant covers `$1`/`$2` (single-digit), which are preserved. Below finding floor.
