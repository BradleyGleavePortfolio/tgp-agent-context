which does not yet exist on `growth-project-backend@main` (only oauth/start,

for the integration PR.

**Impact:** None on this PR's correctness — the mobile slice is complete and
correct against the documented client-side contract, and the path discrepancy
(`/samples/ingest` here vs the controller's referenced `/ingest`) is the
integration PR's to reconcile. Logged so it is not lost.

**Expected fix (integration PR, not this one):** implement the bearer-JWT
`POST` handler accepting `NormalizedSample[]` into the shared `IngestionService`
and reconcile the path with the controller's referenced route.

## Positive observations

- **Native bridge isolation is exemplary:** a single file owns
  `react-native-health`; everything else is typed and unit-testable without a
  device. The local `HealthKitNativeModule` re-declaration insulates the
  connector from upstream `.d.ts` drift.

  advances only after a resolved POST, and never on auth/read/POST failure or an
  empty window — so no samples are silently skipped.
- **Per-metric read fault tolerance:** a single metric's permission denial /
  read failure is surfaced as an omitted field via the `settle()` wrapper rather
  than aborting the whole pass — graceful degradation done right.
- **Unit conversions are defensive and documented** (HRV `<5`, SpO2 `<=1`
  passthrough guards; body-temp deviation baseline documented inline), and the
  canonical enums are verified byte-for-byte against the backend schema.
- **Info.plist strings are decacorn-quality:** they name the specific data and
  the coaching benefit rather than a generic "we use your health data."
