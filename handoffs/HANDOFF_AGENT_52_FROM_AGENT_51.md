## Ingest Envelope Contract (critical — bind this into every C-track PR)

The extension → backend contract is fixed. Do NOT modify without operator sign-off.

```jsonc
POST /api/scout/import
Content-Type: application/json
Authorization: Bearer <supabase_access_token>

{
  "intent_id": "<uuid>",          // idempotency key, unique per (coach_id, intent_id)
  "entity_type": "<string>",       // e.g. "workout", "client", "measurement"
  "entities": [
    {
      "source_id": "<string>",     // stable ID from the source platform
      "payload": {
        "sourcePlatform": "auto:<hostname>",  // e.g. "auto:truecoach.co"
        "capturedAt": "<ISO8601>",             // when the debugger captured this
        // ...arbitrary source-native JSON...
      }
    }
  ]
}
```

- **Idempotency:** replaying the same `intent_id` returns the previously-created ScoutImport IDs with 200. Do NOT re-insert.
- **Tenant boundary:** `coach_id` is derived server-side from the Supabase JWT (`app.current_user_id()` → `User.coach_id`). Never trust a client-supplied `coach_id`.
- **Rate limit:** `@Throttle` per R91.
- **Validation:** DTOs use the global `ValidationPipe(whitelist:true, forbidNonWhitelisted:true)`.
