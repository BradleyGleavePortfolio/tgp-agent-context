## CodeQL workaround assessment
`.github/workflows/codeql.yml` still runs the CodeQL sequence: checkout, `github/codeql-action/init@v3`, `github/codeql-action/autobuild@v3`, then `github/codeql-action/analyze@v3`.

Only the `analyze` step has `continue-on-error: true`. The job itself does not have `continue-on-error`, and init/autobuild do not have `continue-on-error`.

Assessment: this is narrower than a job-level bypass, but it still masks any failure inside `github/codeql-action/analyze@v3`, not only SARIF upload failures caused by disabled GitHub Advanced Security. The check can therefore go green if the actual analyze phase fails for a real reason. This is a doctrine-risking silent fallback rather than a fully valid security gate. At minimum it needs a follow-up guard that distinguishes upload-unavailable from analysis failure, or a documented temporary waiver.
