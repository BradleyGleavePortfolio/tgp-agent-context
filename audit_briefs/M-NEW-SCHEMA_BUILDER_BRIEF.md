## 6. Database changes

Create these tables in this order:

1. `scout_runs`
   - `id uuid primary key default gen_random_uuid()`
   - `coach_id uuid not null`
   - `vendor text not null`
   - `profile_version text not null`
   - `source_mode text not null check (...)` covering `export_upload`, `browser_scout`, `copilot`, `support_assisted`.
   - `status text not null`
   - `started_at`, `finished_at`, `last_heartbeat_at`, `created_at`, `updated_at`
   - `idempotency_key text not null`
   - `audit_correlation_id uuid not null`
   - unique `(coach_id, vendor, idempotency_key)`.

2. `scout_results`
   - Tier-1 user-data table.
   - `id uuid primary key default gen_random_uuid()`
   - `scout_run_id uuid not null references scout_runs(id) on delete cascade`
   - `coach_id uuid not null`
   - `vendor text not null`
   - `entity_type text not null`
   - `vendor_id text null`
   - `source_url text null`
   - `raw_payload jsonb not null default '{}'::jsonb`
   - `parsed_payload jsonb not null default '{}'::jsonb`
   - `payload_hash text not null`
   - `observed_at timestamptz not null`
   - unique `(scout_run_id, entity_type, payload_hash)`.

3. `scout_diffs`
   - Tier-1 user-data table.
   - `id uuid primary key default gen_random_uuid()`
   - `scout_run_id uuid not null references scout_runs(id) on delete cascade`
   - `coach_id uuid not null`
   - `entity_type text not null`
   - `canonical_target_id uuid null`
   - `diff_json jsonb not null`
   - `conflict_resolution text not null`
   - `resolved_by_user_id uuid null`
   - `resolved_at timestamptz null`.

4. `operator_overrides`
   - Mission-critical kill-switch table.
   - `id uuid primary key default gen_random_uuid()`
   - `vendor text not null`
   - `coach_id uuid null`
   - `override_key text not null`
   - `override_value jsonb not null`
   - `reason text not null`
   - `created_by_user_id uuid not null`
   - `created_at timestamptz not null default now()`
   - unique `(vendor, coach_id, override_key)`.
   - Default runtime state when no row exists: `scout_authorized=true`.

5. `session_cookies`
   - Tier-1 secret table.
   - `id uuid primary key default gen_random_uuid()`
   - `coach_id uuid not null`
   - `vendor text not null`
   - `cookie_name text not null`
   - `domain text not null`
   - `encrypted_cookie_value bytea not null`
   - `kms_key_id text not null`
   - `encrypted_data_key bytea not null`
   - `nonce bytea not null`
   - `auth_tag bytea not null`
   - `expires_at timestamptz null`
   - `rotated_at timestamptz null`
   - `created_at timestamptz not null default now()`
   - unique `(coach_id, vendor, domain, cookie_name)`.
   - Cookie values must use envelope encryption: KMS data-key per coach plus per-cookie nonce. Never store plaintext cookies, even transiently in audit state.

6. `scout_audit_events`
   - Thin event table for high-volume, non-PII operational scout events; PII-touching mutations still go through H6A audit log.
   - `id uuid primary key default gen_random_uuid()`
   - `scout_run_id uuid not null references scout_runs(id) on delete cascade`
   - `coach_id uuid not null`
   - `event_type text not null`
   - `event_json jsonb not null default '{}'::jsonb`
   - `created_at timestamptz not null default now()`.

RLS requirements:

- Enable and force RLS on all six tables.
- Coach role: may read/write only rows with `coach_id = current_setting('app.coach_id')::uuid`, except `operator_overrides` is read-only for that coach and platform.
- Operator role: read/write all `operator_overrides`; read scout tables only through explicit operator/admin routes with audit correlation.
- Service role: migration, retention, and erasure only.
- Deny public/anonymous access.

## 7. Test strategy targeting REAL failure modes

Write only tests that catch production failures:

1. `test/scout/schema.rls.spec.ts` — live DB role-switched test proving Coach A cannot read Coach B rows from `scout_results`, `scout_diffs`, or `session_cookies`, and proving coach role cannot write `operator_overrides`.
2. `test/scout/session-cookies.encryption.spec.ts` — verifies inserted cookie rows contain only ciphertext/envelope metadata and no plaintext cookie value in table columns or audit state.
3. `test/scout/operator-overrides.spec.ts` — proves operator role can flip `scout_authorized=false`, coach role can read own platform override, and cross-coach reads remain blocked.
4. `test/scout/migration-reversibility.spec.ts` — verifies `migration.sql` then `down.sql` returns schema objects to baseline once the shared psql URI infra bug is fixed.

If these tests leave ratio <2.0, use `[TEST-EXEMPT: anti-padding-real-failure-modes]` and the R86 block below. Do not add constructor, enum, or snapshot padding.
