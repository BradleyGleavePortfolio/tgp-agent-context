## Audit findings from prior connectors (CRITICAL — bake in from R1)
The Oura/WHOOP/Strava R1 audits surfaced these P1 patterns. Do NOT repeat them.
- **Webhook idempotency ordering:** check `WearableProcessedEvent.upsert` FIRST, then process, then commit. Never mark processed before processing succeeds.
- **KMS token handling:** refresh/access tokens are stored KMS-wrapped via `KmsService`. On read: decrypt → use → discard. On store: encrypt then persist. NEVER write plaintext tokens to DB or logs.
- **PII in logs:** Never log `user_id`, `external_account_id`, raw error response bodies, or raw access tokens. Use `crypto.createHash('sha256').update(user_id).digest('hex').slice(0,16)` for `user_hash` in logs.
- **Zod on webhook payloads:** every webhook controller validates body with a strict Zod schema before any processing. `.strict()` (no passthrough). Invalid → 400 + structured warn log (no payload echo).
- **Fail-closed subscription validation:** if provider returns 5xx/403 on subscription verify during webhook, return 503/403 and DO NOT process. Never silently succeed.
- **Durable enqueue:** if downstream ingestion is async, the webhook controller must persist a `WearableProcessedEvent` row BEFORE responding 200, so a crash doesn't lose the event.
- **OAuth error log redaction:** on `exchangeCode`/`refresh` failure, log only `{ msg, provider, user_id_hash, error_code, error_class }`. NEVER `err.message`, `err.response.data`, or any URL containing tokens.

## Connector interface contract (verbatim — read interface file)
```ts
export type WearableAuthModel = 'oauth2_code' | 'oauth2_pkce' | 'partner_signed' | 'on_device';
export interface RawWebhookRequest {
  headers: Record<string, string | string[]>;
  rawBody: Buffer;
}
export interface WearableConnector {
  readonly provider: WearableProvider;
  readonly authModel: WearableAuthModel;
  buildAuthUrl(userId: string, state: string): string;
  exchangeCode(code: string, opts?: { codeVerifier?: string }): Promise<TokenSet>;
  refresh(conn: WearableConnection): Promise<TokenSet>;
  backfill(conn: WearableConnection, since: Date): Promise<unknown[]>;
  verifyWebhook(req: RawWebhookRequest): Promise<{ valid: boolean; eventId?: string }>;
}
```
